Overview

The MVC, model-view-controller, pattern is a very traditional popular design in the UI, user-interface, and application development.
MOAT IoT provides the ways to enhance the M and the C in your IoT application so that your application can be lightweight, easily developed, and quickly delivered.

Fig. Mapping MOAT IoT to MVC

Models

Regarding Models, under the MOAT IoT concept, each MOAT IoT API allows you to handle the same models even if its underlying architecture is different.
In other words, you have to find and elaborate carefully such the common models through your class design modeling process because the modeling affects all application stacks from the web tier to the device tier.

Fig. Each API spec provides the way to access to essentially identical models
Note that unlike Value Objects (VO), the models should have functions associated with their domain if any.

Models in MOAT REST

Web applications are usually familiar with the JSON representation of models. They are able to marshal and unmarshal objects and JSON texts on invoking MOAT REST APIs. There are much more models in the applications to render web pages. Some of these models are the universal models mentioned here. In addition, the URI of the REST APIs are reflected by the structure design of the models. For example, if your app 'my-app' has a model named 'MyModel' and suppose the model is a singleton, the URI to fetch shall be 'http://host/path/to/moat/v1/my-app/1.0/MyModel'.

Models in MOAT js

Javascript is, of course, good at tweaking JSON representation strings as the name says. Therefore, MOAT js allows you to send/receive JSON texts to the web applications.
With MOAT js, you can get a plain javascript object as the model representation, which is much easier to operate in the javascript code.
MOAT js enables you to invoke a remote method of a model object in a device as well.

Models in MOAT Android and MOAT OSGi Gateway

In your Android/Java application, your Plain Old Java Objects (POJO) are the representations of the models. You can store them to SQLite databases, files or other databases with SQL, O-R mappers

, or whatever.
You can even implement operations to the POJO if you design so.
MOAT Android and MOAT OSGi Gateway certainly propagate the commands of the remote method call from MOAT js implicitly to the POJO with the secured, reliable and proven way.

OMA-DM Capability

If your device is capable of OMA-DM, it is quite easy to integrate it with MOAT IoT applications.
This is because MOAT IoT specifies the conventions to map OMA-DM management objects to MOAT IoT models.
The conventions are easy to learn if you are familiar with recent OMA-DM enabler specifications since they are based upon the design trends of the management objects.
