概要

MVC (Model-View-Controller) は、ユーザインタフェースやアプリケーション開発における、伝統的で有名な設計手段です。
MOAT IoT は、あなたの IoT アプリケーションの Model と Controller を拡張する手段を提供します。ServiceSync 上にアプリケーションを構築することにより、簡単に素早くサービスを展開することができます。

Fig. Mapping MOAT IoT to MVC

モデル(Models)

MOAT IoT のコンセプトにおけるモデルとは、内在するアーキテクチャが異なっていようとも、MOAT IoT API を介して画一的にアプリケーションデータやビジネスロジックを取り扱うことができることを示します。
言い換えると、クラスのモデリングを通して、共通のモデルを、入念かつ注意深く洗い出さなければなりません。なぜなら、モデリングは、Web から デバイスまで全てのアプリケーションスタックに影響するからです。

Fig. Each API spec provides the way to access to essentially identical models
Note that unlike Value Objects (VO), the models should have functions associated with their domain if any.

MOAT REST におけるモデル

Web アプリケーションは、JSON で表現されたモデルと親和性が高いです。MOAT REST API を起動することにより、異なるシステム間でデータを交換することができます。 アプリケーションの世界では、Web ページをレンダリングする多くのモデルが存在します。ここでは、一般的なモデルについて言及します。 加えて、REST API の URI は、モデルの構造設計により左右されます。例えば、あなたのアプリケーション my-app が、MyModel という名前で、かつシングルトンで定義されていると仮定すると、取得の URI の定義は、http://host/path/to/moat/v1/my-app/1.0/MyModel となるでしょう。

MOAT js におけるモデル

Javascriptは、その名の通り、JSON 表現の文字列を扱うのに適しています。従って、MOAT js は、JSON 形式のテキストを Web アプリケーションに送受信することに適していると言えます。
MOAT js を利用すると、モデルをプレーンな Javascript のオブジェクトとして取得することができます。これは、Javascript のコードで簡単にモデルを扱うことができることを示しています。
同様に、MOAT js 介して、デバイス側のモデルオブジェクトのメソッドを起動することもできます。

MOAT Android と MOAT OSGi Gateway におけるモデル

Android や Java アプリケーションでは、Plain Old Java Objects (POJO) が、モデルの表現となります。 モデルは SQLite データベースに保存することもできますし、ファイルに保存することもできます。または、SQL や O-R mappers を利用して、その他のデータベースに保存することもできます。

また、あなたの設計次第で、POJO に対する操作を実装することさえ可能です。
MOAT Android と MOAT OSGi Gateway は、リモートメソッドで起動されたコマンドを、確実にセキュアに、信頼性の高い手段で受信します。

OMA-DM Capability

もし、あなたのデバイスが、OMA-DM を利用可能であれば、MOAT IoT アプリケーションを組み込むのは本当に簡単です。
これは、MOAT IoT が、OMA-DM のマネジメントオブジェクトからMOAT IoT のモデルにマッピングするよう、規約を規定しているためです。
この規約は、最近の OMA-DM Enabler の仕様に詳しければ、習得するのが簡単です。なぜなら、マネジメントオブジェクトの設計トレンドに基づいて設計しているからです。
